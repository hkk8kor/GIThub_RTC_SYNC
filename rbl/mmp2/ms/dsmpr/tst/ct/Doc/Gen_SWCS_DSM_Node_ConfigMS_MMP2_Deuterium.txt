Node is a collection of similar monitorings of a component\sensor\functionality grouped together with definite priority  to determine sequential failures. 
Nodes are vital in deciding the system degradation. Every Node has a parent node and optionally a child node. Only one node in the graph shall existt with no ancestor node.
Network of nodes is called Graph (Failure Dependency structure). 
For each node the following status information are stored:
report of monitoring (suspicion, invalid non recoverable) 
monitoring report while filtered during situation (invalid filtered)
report of curable monitoring (invalid)
not initialized
not available (online variant coding)
The status information of a node is inherit to all child nodes
For each node various static data is stored:
Node mapped to any HWShutdown-
Priority @ Node is disabled
List of all child nodes
List of all assigned MIDs

In the Column RB_Configuration it is indicated for which Mainstream or Release-Branch a item is valid. This document is only used for Deuterium and later releases. Means Beryllium and Carbon is not covered by this document.
Used tags are: 
- Mainsteam_18.3		(object introduced before Deuterium split)
- Mainstream_[BL] 	(current stream development, after Deuterium split)
- Deuterium_[BL] 	(BL indicates object introduced modifed to a Bugfix-Sprint)

Info: the BL version is important, as the test team is evaluating this for component test coverage (CT report / TPR).
The tags shall only be separated by a blankspace.
Last item is Mainstream, if applicable for mainstream.
Empty shall not be used for object in MS_Status "accepted". If it is applicaple for all Release-Branches and Mainsteam all of them have to be listed.

Example:
"Deuterium Mainstream"		--> in Deuterium and Mainstream present
"Deuterium_B03"			--> only in Deuterium with Bugfix Baseline B03 and later
"Mainstream_18.3"		--> only in Mainstream Baseline 18.3 and later

New Release-Branch 
Once a new Release SW is branched out from the Mainstream the Tag in RB_Configuration "Mainstream" has to be replaced by "NewBranchReleaseName Mainstream".

Example:
Before: "Deuterium_B01 Mainstream"  	--> "Deuterium_B01 Europium Mainstream"
Before: "Mainstream_18.3"		--> "Europium Mainstream_18.3"
